### 34. ES 모듈 개념과 내보내기 방식 — export/import, default vs named

#### ES Module 목적

- 파일 단위 코드 재사용

- 스코프 오염 방지

- 명시적 의존성 관리

#### 내보내기 방식

1. Named export

- 여러 개 export 가능

- { } 안에 이름 그대로 import 필요

2. Default export

- 파일당 1개만 가능

- import 시 이름 마음대로 변경 가능

#### import 특징

- 정적 구조 → 번들러가 최적화 가능(Tree-shaking)

35. 동적 import — 지연 로딩과 코드 스플리팅

#### 개념

- import() 형태로 모듈을 필요할 때 비동기로 로딩

- Promise 기반

#### 활용 상황

- 페이지 진입 시 필요한 모듈만 로딩 (코드 스플리팅)

- 무거운 라이브러리를 특정 행동 시 로딩

#### 장점

- 초기 로딩 속도 향상

- SPA에서 성능 최적화
