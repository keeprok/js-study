### 31. 배열 다루기 — map, filter, reduce 활용 방법

#### map

- 배열을 변환하여 새 배열 생성

- 같은 길이 유지

#### filter

- 조건을 만족하는 요소만 추출

- 길이가 변경될 수 있음

#### reduce

- 누적 값을 계산하는 다목적 함수

- 합계, 그룹화, 객체 변환 등 다양하게 활용됨

#### 실무 핵심

- map/filter는 불변성을 유지하기 때문에 React에서 자주 사용됨

- reduce는 데이터 집계/형태 변환에 강력함

### 32. 객체 구조분해 — 가독성과 변수 선언 최적화

#### 목적

- 객체에서 필요한 값만 뽑아내기 쉽게 함.

- 코드 가독성 상승.

#### 예시 상황

- API 응답에서 특정 필드만 사용할 때

- props에서 필요한 값만 추출할 때 (React)

#### 장점

- 중복 코드 제거

- 네이밍 변경 가능 (const { name: username } = user)

### 33. 스프레드 연산자 — 복사/병합의 핵심 문법

#### 배열 복사

- [...arr] → 얕은 복사

#### 객체 복사

- { ...obj } → 불변성 유지

#### 병합

- { ...obj1, ...obj2 }

- 같은 key는 뒤에 오는 값이 덮어씀

#### 실무 중요성

- Redux / React 상태 관리에서 불변성 유지 필수

- 얕은 복사이므로 깊은 중첩 구조는 JSON.parse 또는 structuredClone 필요
